## Resposta #6

**Bruno Gasparetto** (@Bruno_Gasparetto) — 07/10/2022, 10:58 | ❤️ 2

Eu sou muito velho, tá difícil entender essa de programador NoCode, hehehe.

O Fluig, quando chegou na empresa que estou, tinha essa proposta de que qualquer um faria os processos e tal, super simples e “no code”. Mas quando fui mexer realmente descobri que o “no code” não existia e que era uma programação cheia de gambiarra e mega complicada (isso de fazer código JS mega antigo pra virar um Java e não conseguir importar bibliotecas, por exemplo). Várias vezes até conversamos de abandonar e fazer um sistema pra cada processo, pois teríamos muito mais controle, mas pessoal quis continuar testando a ferramenta por mais tempo.

Quando comecei a procurar por Fluig no GitHub achei bastante coisa legal. Ter uma organização é uma boa. Não tenho ideia quem é o dono dessa [fluig · GitHub](https://github.com/fluig), mas se tivesse como juntar tudo num lugar seria maravilhoso.

Eu posto muito mais coisa no meu Gist do que GitHub, pois anoto muita coisa solta (exemplos e orientações) do que projetos completos pra fazer um repositório.
