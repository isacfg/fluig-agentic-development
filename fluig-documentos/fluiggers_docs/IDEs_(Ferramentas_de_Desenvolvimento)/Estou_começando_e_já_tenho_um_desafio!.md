> > Usar VS Code ou Eclipse?

Você será “obrigado” a usar o Eclipse ao menos para exportar os itens para o Fluig, utilizando a extensão do Fluig e usando o Eclipse homologado pra não ter problemas.

Mas você pode desenvolver a parte Web em qualquer IDE/editor e usar o Eclipse só pra exportar/importar os artefatos. Já o diagrama BPM acredito que só dê pra fazer no Eclipse mesmo.

Até existem algumas extensões não oficiais de Fluig para o VSCode, mas elas não fazem tudo que o Eclipse faz.

## Resposta #3

**Bruno Gasparetto** (@Bruno_Gasparetto) — 10/02/2022, 11:21 | ❤️ 1

Eu sou mais um “curioso” de Fluig. Acabo desenvolvendo algumas coisas por ser o gestor do contrato do Fluig aqui na empresa e ter experiência com programação. Mas eu prefiro usar o VS Code pra ter autocompletar melhor e tal. Aí uso o Eclipse pra exportar os artefatos só. Como eu não passo o dia mexendo com programação no Fluig pra mim isso é melhor, mas não sei como seria com os demais.

Pelo que lembro o Fluig já tem um processo de tarefas bem simples, mas não lembro como funciona pois o desabilitamos para não ter perigo de criarem tarefas nele ao invés dos processos específicos.

Aí cabe a você analisar esse repositório que encontrou e analisar se te atenderia ou se você vai precisar criar um processo do 0.

Pessoal que trabalha com desenvolvimento Fluig logo deve dar uma opinião tbm :slight_smile:
