# Exportando processos

> **Fonte:** [https://tdn.totvs.com/display/fluig/Exportando+processos](https://tdn.totvs.com/display/fluig/Exportando+processos)

---

Ao modelar um processo, é possível verificar se existem inconsistências antes de exportá-lo para o servidor fluig. Por exemplo, um mecanismo de atribuição por grupo pode estar configurado sem existir um grupo selecionado. Em caso de erro, um X em vermelho é exibido na atividade e no ícone do processo.

Não é possível exportar o processo para o servidor até que todos os erros da modelagem sejam corrigidos.

Para exportar um processo para um servidor fluig, basta clicar no processo com o botão direito, e selecionar a opção **Export...**

Na janela de exportação, escolher a opção **Exportar para um Servidor Fluig** e clique em **Next**

É possível também exportar o processo para os seguintes formatos:

-   Formato Site - Irá gerar uma página web com todas as informações sobre o processo
-   Formato XPDL - Criará um arquivo de extensão .xpdl que pode ser importado em outros softwares de modelagem

Neste último passo, escolhemos o servidor destino do processo. O plugin irá carregar os processos já cadastrados no servidor e, caso encontre o processo que está sendo exportado, o usuário poderá atualizar o modelo existente.

Marcando a opção de **Novo Processo** o plugin irá criar um processo com o nome informado. Após preencher os dados, clique em **Finish** e espere o procedimento ser finalizado.
