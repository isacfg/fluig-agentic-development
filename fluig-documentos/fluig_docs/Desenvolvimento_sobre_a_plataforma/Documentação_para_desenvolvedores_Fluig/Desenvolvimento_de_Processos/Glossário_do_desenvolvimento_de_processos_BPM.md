# Glossário do desenvolvimento de processos BPM

* * *



Quando se trata de gerenciamento de processos de negócios, é importante estar familiarizado com alguns termos-chave. Neste glossário, explicaremos brevemente alguns dos termos que serão usados ​​na documentação relacionada ao Fluig BPM. Para os termos que precisam de uma explicação mais específica, criamos uma página que pode ser acessada clicando no termo.

### **Processo (Workflow)**

É um conjunto de atividades que devem ser executadas de forma sequencial para atingir um objetivo específico (Saiba mais). 

### **Solicitação**

É a instância de um processo, ou seja, é cada utilização do processo (Saiba mais).

### **Formulário**

É uma ferramenta usada para a entrada de informações pelos usuários. Ele pode estar associado a um processo ou mesmo ser um pré-cadastro de fornecedores, clientes ou outros conjuntos de dados que serão usados em outros formulários.

### **Registro de formulário**

Cada preenchimento/resposta de formulário gera um novo documento que chamamos de Registro de Formulário.

### **Dataset**

É um conjunto de dados que podem ser utilizados por processos e formulários. Estes dados podem ser desde listas fixas onde os dados são criados um a um via programação, passando pelos registros de formulários do fluig, ou como ponto de contato com outros sistemas ou banco de dados permitindo inclusive o armazenamento dos dados para o uso sem que o tempo de resposta da integração atrase a movimentação da tarefa dos usuários..

### **Tarefa**

É uma atividade específica que deve ser concluída durante a solicitação.

### **[Thread](https://tdn.totvs.com/display/fluig/Thread+de+Processo)**

Sequencial para o controle de paralelismo, ou seja, quando um processo possuí um gateway do tipo fork ou inclusivo.

### **Mecanismo de atribuição**

É o processo pelo qual as tarefas são atribuídas a usuários específicos. O mecanismo de atribuição pode ser simples ou complexo, dependendo do processo.

### **Personalização**

É a ação de personalizar o Fluig BPM para atender às necessidades específicas da empresa. 

### Termos técnicos

**[Case Sensitive / Case Insensitive](https://tdn.totvs.com/pages/viewpage.action?pageId=754248018)**

É a indicação se um método ou função faz a diferenciação entre letras maiúsculas e minúsculas.
